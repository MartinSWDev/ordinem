04 · Build Spec Point every Vue component here. Copy the prompt below.
✓ Always
  • Element background = its container background (var(--surface)). Neumorphism only reads when they match.
  • Raised for interactive/grouped; inset for wells, inputs and pressed.
  • Orange = primary action or live/active state only.
  • Depth from shadow, hierarchy from spacing & type — not borders.
  • Icons: 1.5–2px stroke, line style, currentColor.
  • Every color/shadow/radius via a CSS var — never hard-coded.
✕ Never
  • Hard drop-shadows, pure black, or heavy borders as separators.
  • More than one accent hue, or orange used decoratively.
  • Elements floating on a background that isn't the surface color.
  • Gradients, glassmorphism, or filled colored cards.
  • Rely on shadow alone for state — pair with color/icon for a11y.
  • Nesting raised-inside-raised too deep; keep to 2 elevation levels.
Component build prompt 
Build a Vue 3 component for the Ordinem dashboard — a neumorphic, mission-control ("NASA science") macOS app. Follow these rules exactly.

STYLE
- Soft-UI / neumorphism, SUBTLE depth. Shadow, not borders, creates elevation.
- The component's background MUST equal its parent's (--surface). This is what makes neumorphism work.
- Two elevation tools only:
    raised  → box-shadow: var(--shadow-out)   (interactive / grouped)
    inset   → box-shadow: var(--shadow-in)     (wells, inputs, pressed, tracks)
- Never exceed two nested elevation levels.

TOKENS (theme via CSS custom properties; light theme — dark deferred)
  --surface:  #E7EAF0  (light)   /  #24272E (dark)
  --text:     #2A2E36           /  #E7E9ED
  --text-dim: #7E8794           /  #8A919C
  --accent:   #F26A1B  (both modes)
  --success:  #2E9E5B
  --sh-dark:  #C4C9D3           /  #131519
  --sh-light: #FFFFFF           /  #2C3036
  --sh-dist: 5px; --sh-blur: 11px; --sh-din: 4px; --sh-blin: 8px;
  --shadow-out: var(--sh-dist) var(--sh-dist) var(--sh-blur) var(--sh-dark),
                calc(var(--sh-dist)*-1) calc(var(--sh-dist)*-1) var(--sh-blur) var(--sh-light);
  --shadow-in:  inset var(--sh-din) var(--sh-din) var(--sh-blin) var(--sh-dark),
                inset calc(var(--sh-din)*-1) calc(var(--sh-din)*-1) var(--sh-blin) var(--sh-light);
  radii:  --r-lg 10px (cards) · --r-md 7px (controls) · --r-sm 4px  (SHARP corners)
  app icon: solid --accent squircle (23% radius) + white ō glyph @ 56%
  space:  4 / 8 / 12 / 16 / 24 / 32

TYPE
  Display : 'Space Grotesk', 600–700, letter-spacing -0.02em (titles)
  Body/UI : 'IBM Plex Sans', 400–600, 13–15px
  Data    : 'IBM Plex Mono', uppercase micro-labels, letter-spacing .14em, 10–11px

COLOR
- Orange = primary action OR live/active state ONLY. Everything else is grey ink on surface.
- Success green only for passing/added states.

INTERACTION
- hover: lift slightly (increase --sh-dist ~1px) ; active/pressed: swap to var(--shadow-in).
- focus: 1.5px solid var(--accent) ring; keep the inset well.
- Transition box-shadow & transform 160ms ease.

A11Y
- Never signal state with shadow alone — pair with color + icon/label.
- Maintain 4.5:1 text contrast; --text-dim only for secondary text ≥12px.

DELIVER
- A single-file Vue 3 SFC, scoped styles, props typed, tokens read from CSS vars (never hard-coded).
- Emit standard events; expose size/variant via props (variant: primary|secondary, size: sm|md).
